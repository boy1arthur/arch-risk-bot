# Contributing

프로젝트 기여를 환영합니다.

## Ground Rules

- 변경은 작은 단위로 나누어 커밋합니다.
- 커밋 메시지는 의도를 명확히 작성합니다.
- 관련 문서(README/docs)도 함께 갱신합니다.

## Development Flow

1. 이슈를 생성하거나 기존 이슈와 연결합니다.
2. 브랜치를 생성합니다.
3. 변경 후 테스트/검증을 수행합니다.
4. Pull Request를 생성합니다.

## Pull Request Checklist

- [ ] 변경 목적이 PR 본문에 명확히 설명되어 있음
- [ ] 문서가 필요한 경우 함께 업데이트함
- [ ] 불필요한 파일/민감정보가 포함되지 않음
