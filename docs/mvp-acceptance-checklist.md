# ✅ MVP Acceptance Test Checklist

Arch Risk Bot MVP 버전의 안정성을 검증하기 위한 최종 체크리스트입니다.

## 1. Core Functionality (핵심 기능)
- [ ] **PR 감지**: 새로운 PR이 생성되면 30초 이내에 분석이 시작되는가?
- [ ] **코멘트 생성**: 분석 완료 후 PR 하단에 진단 코멘트가 정상적으로 작성되는가?
- [ ] **제안 품질**: AI가 제안한 리팩토링 패치가 구문적으로 올바른 Python 코드인가?

## 2. Safety & Guardrails (안전 장치)
- [ ] **Suggestion-Only 보장**: 봇이 코드를 직접 수정하거나 커밋을 생성하지 않는가?
- [ ] **자동 수정 고지**: 코멘트 상단에 "자동 변경 없음/제안 전용" 문구가 명확히 표시되는가?
- [ ] **규모 제한**: 파일 20개 또는 2,000라인 초과 시 "Summary-only" 경고가 출력되는가?
- [ ] **Ignore 규칙**: `node_modules` 등 제외 폴더 내의 파일은 분석에서 제외되는가?

## 3. Resilience (복구력)
- [ ] **에러 핸들링**: 분석 중 오류 발생 시 Crash 되지 않고 로그를 남기며 종료되는가?
- [ ] **타임아웃**: 개별 파일 분석이 10초를 초과할 경우 안전하게 중단되는가?
- [ ] **API 레이트리밋**: GitHub 또는 Gemini API 제한 도달 시 사용자에게 적절한 안내가 나가는가?

## 4. UI/UX (사용자 경험)
- [ ] **링크 정확도**: 코멘트 내의 파일/라인 링크가 실제 PR의 Diff 뷰로 연결되는가?
- [ ] **가독성**: 템플릿 v1(요약-리스크-근거-제안)이 모바일/데스크톱 GitHub에서 깨짐 없이 보이는가?
